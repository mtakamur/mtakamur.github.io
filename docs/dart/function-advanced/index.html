関数

Dart における関数定義のまとめ。 基礎編の内容を前提としています。 また、解説中では null safetyが有効である想定です。

記法のバリエーション

全く同じ関数であっても、その定義の記法にはバリエーションがあります。 下記 4 つの例は、いずれも全く同じ動作をするsayHello関数です。

String sayHello() {
  return "Hello";
} // (1.1)

sayHello() {
  return "Hello";
} // (1.2)

String sayHello() => "Hello"; // (1.3)

sayHello() => "Hello"; // (1.4)

(1.1)の記法は、基礎編でも紹介した最も基本的な記法です。 戻り値の型を明記したうえで、returnによって値を返しています。

(1.2)の記法は、(1.1)から戻り値の型を省いた記法です。 変数定義において働く型の推論とは異なり、関数の戻り値の型を省略した場合には型は推測されず、常にdynamicなインスタンスを返す関数とみなされることに注意が必要です。 Effective Dartでも紹介されている通り、戻り値の型が静的に決められる場合には dynamic 型の戻り値にする理由がありませんので、多くの場合で(1.2)の記法は避けるほうが無難です。 また、戻り値の型を dynamic にする必要があるのであれば、下記のようにその旨を明記するべきです。

dynamic sayHello() {
  return "Hello";
}

可能な限り戻り値の型は静的に確定しましょう。

(1.3)および(1.4)はこの節で初めて紹介する書き方で、関数の処理本体が 1 行に収まる場合に用いることができる省略記法です。 個の記法では、関数の処理本体を囲む {} および関数の終点を表す return を省くことができるため、単純な関数をコンパクトに記述することができます。 returnを直接記述するイメージです。 それぞれ(1.1)と(1.3)、(1.2)と(1.4)は完全に等価です。

関数のパラメータ

関数の処理内で必要になる情報を関数の外部から受け取る必要がある場合があります。 関数はこのような情報をパラメータ(引数)として関数の呼び出し元から受け取ることができます。

基本形

関数でパラメータを受け取る基本形は次のようになります。

void main() {
  var result = double(2);
}

int double(int seed) => seed * 2; // (2.1)

上記の double 関数はパラメータとして受け取った整数を 2 倍して返す関数です。 int seed の部分がパラメータの定義で、int 型のパラメータを受け取って、その値を double 関数内で seed という名前で利用することを表しています。 double 関数を main から呼び出しており、2 という値を double 関数に与えています。 double の呼び出し個所を見てわかる通り、呼び出し側はパラメータがどんな名前で関数内で利用されるかは見えない(double 関数内で seed という名前で利用されることを知らない)ようになっています。 (2.1)の関数定義では、パラメータ seed は必須のパラメータとして扱われるので、呼び出し時にパラメータの入力を省略することはできません。

Named parameter (名前付きパラメータ)

前述の基本形の記法では、関数の内部ではパラメータは名前付きの変数として利用することができますが、関数の呼び出し元からはどんな意味のパラメータを与えているのかは定義を見ないとわかりません。

関数のパラメータの数が一つや二つであれば問題を感じないかもしれませんが、同じ型のパラメータが複数必要になる場合など、呼び出し元からもどんな名前のパラメータを入力しているのかが見えると便利な場合があります。

以下の Named parameter の記法を用いると、関数の呼び出し元でパラメータの名前を指定して値を入力することができるようになります。

void main() {
  var returnedMessage = say(message: "Hello World"); // (2.2)
}

String say({String? message}) { // (2.3)
  return "Fake message";
}

(2.3)が Named parameter の記述ですが、パラメータの定義一式を {} で囲むだけですが、下記に注意する必要があります。

  • Named parameter はデフォルトで optional parameter となる
  • Named parameter を 必須パラメータにしたい場合には、required であることを明記する必要がある

Named parameter を設定すると、これはデフォルトで optional のパラメータとなります。 つまり、呼び出し元で必要に応じてパラメータ入力を省くことができるようになります。 パラメータ入力が省かれた場合は、対応するパラメータが存在しないことになりますので、パラメータの型を nullable にするか、パラメータのデフォルト値を設定しておく必要があります。 パラメータのデフォルト値の設定については後述します。

Named parameter が定義された関数を呼び出す場合は(2.2)のように パラメータ名: 値 の形式で列挙して入力します。

関数のパラメータが Named parameter によって定義されていると、基本形のように呼び出し元でパラメータ名を省いた呼び出しができなくなりますので注意してください。

余談ですが、下記のようにパラメータの一部分だけを Named parameter によって定義することも文法上は可能です。 ただし、(2.4)を見てわかる通り呼び出し方がかなりへんてこな形になりますので、避けたほうが無難でしょう。 関数は基本形のまま、パラメータの一部を optional にしたい場合には以下で説明する optional position parameter を用いることができます。

void main() {
  introduceHuman("umi", age: 0); // (2.4)
}

void introduceHuman(String name, {int? age}) =>
    print("Name = $name, age = $age");

Optional position parameter

Position parameter というとピンと来ないかもしれませんが、実は基本形の関数のパラメータも Position parameter です。 Named parameter と異なり、関数呼び出し時に対応するパラメータ名を指定することはできず、入力値とパラメータの対応関係は入力順 (= position) によって結び付けられるためこう呼ばれています。

Position parameter による関数定義と呼び出しは以下のようになります。

void main() {
  var message = say(); // (3.1)
}

String say([String? message, String suffix = "."]) { // (3.2)
  return "$message$suffix";
}

Named parameter を定義するのに {} を用いたのに対して、optional position parameter を定義するには(3.2)のように [] を用います。 パラメータの nullability のルールはもちろん Named parameter の場合と同じです。 パラメータを nullable にするか、デフォルト値を設定しておく必要があります。

こうすると(3.1)にあるように、パラメータ入力を省いて関数を呼び出すことができます。

Named parameter と異なり、パラメータ名をいちいち入力する必要がない代わりに一つ困ったことがあります。 入力順がそのままパラメータに対応するため、例えば(3.2)のメソッドであれば message の入力を省いて suffix のみ入力するということができません。 そのため、必須パラメータはパラメータ前方へ、optional パラメータは後方にまとめて定義するなど、順序の工夫が必要になります。

void sample(int mandatory, [int? optional1, int? optional2]) {}

必須、optional の違いによってパラメータの定義順がある程度決められてしまうなど、本質的でない制約が加わることになるため、Named parameter が利用できる Dart において optional position parameter の利用が最善の選択となる場面はそう多くないように思います。

optional なパラメータが必要な関数では、デフォルト値を伴う Named parameter によって、意味のある順番でパラメータを定義するのが良いでしょう。

パラメータのデフォルト値

すでに何度か登場していますが、パラメータの入力が省かれた際、そのパラメータの値として null ではなく、特定の値を設定したい場合があります。

String say({String message = "Hello World!"}) { // (4.1)
  return message;
}

パラメータにデフォルト値を設定できるのは、そのパラメータが Named parameter もしくは optional position parameter である場合のみです。 基本形のパラメータは呼び出し側での入力が必須となるので、関数の定義側でデフォルト値を設定しておく意味がないのです。

上記は Named parameter に対してデフォルト値を設定している例です。 デフォルト値定義の良いところは、optional パラメータを必要以上に nullable にせずに済むことです。 もちろん必要な場面で null を用いるのは問題ないですが、単にパラメータの無入力状態を表すために null を用いるのは多くの場合 null の乱用になります。

オブジェクトとしての関数

Dart では関数もオブジェクトとして扱います。 従って、メソッドによってはパラメータとして関数そのものを渡すことができる場合があります。

例えば、コレクションクラスに実装されている forEach メソッドは、 int を受け取る void 関数 を受け取ることができます。 従って、下記のように list.forEach に対して printInt をパラメータとして渡すことができます。

void main() {
  var list = [1, 2, 3];
  list.forEach(printInt);
}

void printInt(int element) {
  print(element);
}

ちなみに、関数をパラメータとして受け取る関数は以下のように記述します。

void main() {
  printResult(seed: 1, calculation: doubleSeed);
}

double doubleSeed(int seed) => seed * 2.0; // (5.1)

void printResult({required int seed, required double calculation(int seed)}) { // (5.2)
  print(calculation(seed));
}

肝は(5.2)の部分です。 printResult の第二パラメータ calculation(int seed) は int 型のパラメータを 1 つ受け取り double 型の値を返す関数を表します。

(5.1)の doubleSeed が上記の条件に合致する関数です。 main 関数の中で、 printResult に対して doubleSeed 関数をパラメータとして与えています。

printResult 関数の内部では、 パラメータとして与えられた calculation は int を受け取り double を返す通常の関数として利用することができます。

無名関数

上記のようにパラメータとして関数を渡す場合、関数を通常の変数のように扱えると直感的に扱える場合があります。 その際に利用できるのが無名関数 (Anonymous functions) です。

  var amplifier = (int seed, int gain) => seed * gain; // (6.1)
  var amplifier = (int seed, int gain) {
    return seed * gain;
  }; // (6.2)

上記(6.1), (6.2)は等価な無名関数定義です。 ともに int 型のパラメータを 2 つ受け取って、int 型の計算結果を返却する関数を定義しています。

ぱっと見関数に amplifier という名前を付けているように見えるため、なぜ無名関数と呼ばれているのかわかりづらいかもしれません。 ポイントになるのは、変数はあくまで特定の型のインスタンス(への参照)を格納するための箱だということです。 つまり、 amplifier はあくまで関数を格納するための箱の名前であり、関数の処理自体に名前を与えているわけではないということです。 Dart における変数は変更可能 (mutable) なため、変数 amplifier に別の関数を格納すると、元の処理は完全に参照を失うことになります。 これは、通常の関数定義のように関数事態に命名されていればあり得ないことです。 これが無名関数たる所以です。

ちなみに、(6.1),(6.2)とも戻り値の型を定義していませんが、戻り値は int 型と推論されます。 戻り値の型の定義がなくとも、通常の関数と異なり dynamic 型を返す関数にはなりません。 多くの場合問題にはなりませんが、この振る舞いは理解しておく必要があります。
