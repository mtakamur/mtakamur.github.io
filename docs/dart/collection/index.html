コレクション

Dart におけるコレクション(配列)の扱いを学びます。

いくつかの要素を集合として扱うためのオブジェクトをコレクションと呼びます。 Dart では、コレクションの型として以下の 3 種類が用意されています。

  • List: 重複を許す単一の型の要素の集合
  • Set: 重複を許さない単一の型の要素の集合
  • Map: 値に対して特定のキーを結び付けて参照する集合

List

Listは、追加された順番通りに重複を許して要素を保持するコレクションです。 以下のように、list literal []によって生成します。

  List<String> wordsList = ["hoge", "huga"];
  var wordsList = ["hoge", "huga"]; // 上記と等価

リストは要素として様々な型を格納することができます。 そのため、リストを変数として宣言するときには何の型のリストなのかを明記する必要があります。 これをジェネリクスと呼び、利用する型パラメータの指定は<>によって行います。 例えば上記では、型パラメータとしてStringを指定しているので、生成されるリストはString型を格納するためのリストになります。

リストに格納される要素の型は単一(Stringとintを同じリストに格納はできない)なので、要素の型から一意にリストの型パラメータが推測できます。 そのため、 多くの場合varによって型推論に頼って生成することも可能です。

Set

Setの生成は下記のように行います。 Listと作り方はほとんど同じですが、要素を列挙する際 set literal {} を用います。 ジェネリクスを持つため、一般に型パラメータの指定が必要な点はListと同じです。

  Set<String> set = {"this", "is", "a", "set"};

ListとSetはともに同じ型の要素を複数まとめて扱うコレクションですが、Setは以下の点でListとは異なります。

  1. 順番の概念がない
  2. 同じ要素を複数持たない

順番の概念がない

前述のとおりListは要素が追加された順番通りに要素が保持されますが、Setは順番の概念がありません。 Setはiterableを継承しているので、最初や最後の要素にアクセスするためのメソッド、firstやlastが利用できますが、順番は保証されていませんのでSetにおいてこれらのメソッドは利用するべきではありません。 Setの要素に対して操作が必要な場合には、iterator を取得してイテレーティブに操作することが必要となります。

重複する要素を持たない

Setは重複する要素を複数保持しません。 例として、下記のようにSet<String>に対して同じ文字列を追加してみます。

void main() {
  Set<String> set = {"value"};
  set.add("value");
  print(set);
}

実行結果は次のようになります。 setに"value"を 2 つ追加しているはずですが、結果は"value"が 1 つだけ含まれていることがわかります。 String型のセットの例を示しましたが、どの型についても重複を許さないのは同じです。

{value}

Map

Mapはコレクションの一つで、key と value のペアで値を保持します。 Listでは要素ごとに自動的に与えられる index を値に結び付けていますが、Mapでは値に任意の key を結び付けることができます。 ちょうど各 value に対して名前を付けてあげるイメージです。 Mapでは、key と value それぞれの型パラメータを指定する必要があります。

下記の例では、key の型と value の型をともにString型に指定しています。 お気づきかもしれませんが、Mapの生成はSetを作るときに使った set literal {}を用います。 key と value を : で区切り、key-value ペアを列挙していきます。

    Map<String, String> map = {"key": "value"};

Setの生成と用いる記法が共通していることで、1 つ注意しなくてはいけないことがあります。 型推論によって生成された、以下のインスタンスambiguousの型は何になるでしょうか？

  var ambiguous = {};

実は、ambiguousの型はMap<dynamic,dynamic>となります。 ぱっと見では空のSetを生成しているように見えるかもしれませんが、実際には key、value ともにdynamic型のMapが生成されます。 空の set literal {} では SetではなくてMapが作られることに注意してください。
